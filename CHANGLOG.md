# CHANGELOG

## [0.5.2] - 2026-04-13

### Added
- Supabase `project URL`과 `publishable key`를 사용하는 프론트엔드 인증 연결을 추가했다.
- 기본 회원가입, 로그인, 로그아웃 기능을 추가했다.
- 화면 오른쪽 위에 고정되는 계정 패널 UI를 추가했다.

### Changed
- 시작 오버레이를 게임 시작 안내 중심으로 다시 정리했다.
- 로그인한 계정이 있으면 닉네임 안내가 계정 정보 기준으로 동작하도록 조정했다.
- 로그인하지 않은 상태에서는 게임을 시작할 수 없도록 변경했다.
- 기존 브라우저 랭킹 저장 로직이 로그인 계정 식별자를 함께 활용하도록 바꿨다.

### Notes
- 현재 인증은 Supabase 기본 provider 기준으로 연결되어 있다.
- 랭킹은 아직 Supabase DB가 아니라 브라우저 `localStorage`에 저장된다.

## [0.5.3] - 2026-04-13

### Added
- Supabase `rankings` 테이블 생성용 SQL 파일 `supabase/online-ranking.sql`을 추가했다.

### Changed
- 랭킹 저장 방식을 브라우저 `localStorage`에서 Supabase 온라인 랭킹으로 전환했다.
- 게임 오버 시 점수 저장이 비동기로 처리되도록 바꿨다.
- 랭킹 패널이 Supabase에서 실시간으로 상위 점수를 불러오도록 변경했다.

### Notes
- 온라인 랭킹 사용 전 `supabase/online-ranking.sql`을 Supabase `SQL Editor`에서 실행해야 한다.
- 현재 구조에서는 `Secret key` 없이 `publishable key`와 RLS 정책만으로 온라인 랭킹을 처리한다.

## [0.5.4] - 2026-04-13

### Changed
- 인증 방식을 사용자에게 보이는 기준으로 `이름 / 아이디 / 비밀번호` 흐름으로 재구성했다.
- 메인 화면 우측 상단 인증 UI를 작은 `로그인` 버튼 중심의 접이식 패널로 변경했다.
- 로그인 패널 하단의 `회원가입` 버튼을 눌러 가입 모드로 전환되도록 바꿨다.

### Notes
- Supabase 인증은 사용자 기준으로 `이름 / 아이디 / 비밀번호` 흐름만 보이도록 정리했다.
- 사용자 화면에는 이메일 입력이 노출되지 않는다.

## [0.5.5] - 2026-04-13

### Changed
- 사용자 화면에서 이메일 관련 표현과 안내 문구를 추가로 정리했다.
- 로그인한 계정 이름 표시가 `이름(아이디)` 형태로 더 자연스럽게 보이도록 다듬었다.
- 내부 인증용 식별자 생성 함수와 관련 명칭을 사용자 중심 표현에 맞게 정리했다.
- 앱 시작 시 예전 브라우저 `localStorage` 랭킹 데이터를 자동으로 정리하도록 추가했다.

### Notes
- Supabase Auth 구조상 내부적으로는 아이디 기반 숨김 주소를 계속 사용한다.
- 사용자에게 보이는 회원가입과 로그인 흐름은 계속 `이름 / 아이디 / 비밀번호` 기준이다.

## [0.5.6] - 2026-04-13

### Changed
- 시작 화면의 닉네임 입력란을 제거했다.
- 온라인 랭킹 저장 이름을 닉네임 대신 로그인한 아이디로 고정했다.
- 게임 시작 안내 문구를 아이디 기반 랭킹 저장 흐름에 맞게 정리했다.

## [0.5.7] - 2026-04-13

### Changed
- 회원가입과 로그인에서 한국어 아이디를 사용할 수 있도록 아이디 정규화 규칙을 확장했다.
- 내부 인증용 식별자를 아이디 문자열 직접 사용 대신 해시 기반 값으로 바꾸어 한글 아이디도 안정적으로 처리되게 했다.
- 로그인 입력란 안내 문구를 한글 아이디 지원 기준으로 업데이트했다.
- 기존 영문 아이디 계정은 계속 로그인되도록 이전 인증 방식과의 호환성을 유지했다.
